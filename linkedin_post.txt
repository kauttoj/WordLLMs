

What I learned from this project:

-multiagent collaborations are very difficult to orchestrate and control. When  agent  are let freely interact and discuss, results can be anything between chaos and brilliance. Despite writing careful instructions, variability how things proceed remains huge. When possible, work with one capable LLM or agent which allows you to guide the process after each turn. If you allow multiple LLMs take turns and react to each other freely, they can  easily steer off the course.
Note that this does not apply situations where LLMs/agents are working separately or part of a deterministic workflow where their outcomes are controlled. 

-orchestrating different LLMs (Gemini, Claude and OpenAI) can be pain in the ass. Each vendor have some quirks and restrictions how messages must be passes and how tools are uses. For example, Claude requires strict "user" and "assistant" messages alternation; you must make last message from user when asking a response. Gemini does not error, but silently start acting badly.

-handling context over long discussion with different actors, tools and attachments involved must be done carefully. You constantly run risk of mixing roles ("who said what"), tool calls/results and context rotting.

-Balancing between dozens of different agent tools requires careful prompting. When to use tool X instead of slightly similar tool Y? When to give up after tool Z won't work?

-Open-source developer is seriously outnumbered and in a subordinate position when developing tool for a proprietary software. Microsoft has full power over what developers can do with Office, this case Office.js component. If they decide to remove some core function (or not add some necessary one), it can easily kill projects. For example, with Word there is no official tools to make edits under text revisions or doing multi-part text selections.


So nothing new under the sun for developers, but a good reminder.
